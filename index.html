Documentação Completa de Comandos Linux e Ferramentas de Cybersegurança

Bem-vindo à documentação abrangente de comandos Linux e ferramentas de cybersegurança. Este recurso cobre desde comandos básicos do sistema Linux até ferramentas avançadas de segurança digital usadas por profissionais de cybersegurança.

Propósito Educacional

Esta documentação foi criada exclusivamente para fins educacionais e de aprendizado ético. As ferramentas e técnicas apresentadas aqui devem ser usadas apenas em ambientes controlados, em seus próprios sistemas ou com autorização explícita. O uso indevido dessas ferramentas pode constituir crime em muitas jurisdições.

Sobre o Linux

O Linux é um sistema operacional de código aberto baseado em Unix. É conhecido por sua estabilidade, segurança e flexibilidade. O que distingue o Linux é sua interface de linha de comando (CLI) poderosa, que permite aos usuários executar tarefas complexas com comandos de texto simples.

Sobre as Ferramentas de Cybersegurança

Ferramentas de cybersegurança são utilizadas por profissionais para identificar vulnerabilidades, testar a segurança de sistemas e redes, e proteger infraestruturas digitais. Muitas destas ferramentas estão incluídas em distribuições Linux especializadas como o Kali Linux, que é desenvolvido especificamente para testes de penetração e análise de segurança.

Se você está começando, recomendamos primeiro se familiarizar com:

  • Comandos básicos de navegação (cd, ls, pwd)
  • Manipulação de arquivos (cp, mv, rm)
  • Visualização de arquivos (cat, less, more)
  • Conceitos básicos de permissões (chmod, chown)

Concentre-se em entender a estrutura do sistema Linux e como navegar pelo terminal antes de avançar para as ferramentas de segurança.

Para usuários com conhecimento básico de Linux, explore:

  • Manipulação de texto avançada (grep, sed, awk)
  • Gerenciamento de rede (ifconfig/ip, netstat, ssh)
  • Shell scripting básico
  • Ferramentas básicas de reconhecimento (Nmap, Whois)
  • Monitoramento de sistema (top, htop, ps)

Para usuários avançados e profissionais de segurança:

  • Ferramentas complexas como Metasploit, Wireshark e Burp Suite
  • Scripting avançado para automação de tarefas de segurança
  • Técnicas de evasão e ofuscação
  • Análise forense digital
  • Engenharia reversa de malware
  • Testes de penetração completos

Permissões

No Linux, as permissões de arquivos são baseadas em três grupos: proprietário (owner), grupo (group) e outros (others). Cada grupo tem três tipos de permissões: leitura (r), escrita (w) e execução (x).

chmod 755 arquivo.sh # Define permissões (rwxr-xr-x)
chown usuario:grupo arquivo.txt # Altera proprietário e grupo
Notação Numérica de Permissões

As permissões no Linux podem ser representadas por números:

  • 4: Permissão de leitura (r)
  • 2: Permissão de escrita (w)
  • 1: Permissão de execução (x)

Combinando esses valores, criamos as permissões para cada grupo:

  • chmod 755 = rwxr-xr-x (proprietário: tudo; grupo e outros: ler e executar)
  • chmod 644 = rw-r--r-- (proprietário: ler e escrever; grupo e outros: apenas ler)
  • chmod 777 = rwxrwxrwx (todos podem ler, escrever e executar - geralmente inseguro)

Shell Scripting

Shell scripting é uma poderosa forma de automatizar tarefas no Linux, combinando comandos e lógica de programação em arquivos executáveis.

Bash Scripting

Bash (Bourne Again SHell) é o shell padrão em muitas distribuições Linux e oferece uma linguagem de scripting robusta.

Estrutura Básica de um Script Bash

Todo script bash deve começar com o shebang, que indica qual interpretador deve ser usado:

#!/bin/bash

# Isto é um comentário
echo "Olá, Mundo!"

# Terminar com status de saída 0 (sucesso)
exit 0

Para tornar um script executável:

chmod +x script.sh # Torna o script executável
./script.sh # Executa o script

Variáveis e Condicionais

Bash permite declarar variáveis e usá-las em scripts:

#!/bin/bash

# Variáveis
NOME="Usuário"
NUMERO=42

# Usar variáveis
echo "Olá, $NOME!"
echo "Seu número é $NUMERO"

# Variável de ambiente
echo "Seu diretório home é $HOME"

# Substituição de comando
DATA=$(date +%Y-%m-%d)
echo "A data atual é $DATA"

# Condicional if-else
if [ $NUMERO -eq 42 ]; then
echo "O número é 42!"
elif [ $NUMERO -gt 42 ]; then
echo "O número é maior que 42!"
else
echo "O número é menor que 42!"
fi

# Testando arquivos
if [ -f "/etc/passwd" ]; then
echo "O arquivo /etc/passwd existe."
fi

if [ -d "/var/log" ]; then
echo "O diretório /var/log existe."
fi
Operadores de Comparação Numérica
  • -eq: igual
  • -ne: não igual
  • -gt: maior que
  • -lt: menor que
  • -ge: maior ou igual
  • -le: menor ou igual
Operadores para Arquivos e Diretórios
  • -f arquivo: existe e é um arquivo regular
  • -d diretorio: existe e é um diretório
  • -e caminho: existe (qualquer tipo)
  • -r arquivo: existe e tem permissão de leitura
  • -w arquivo: existe e tem permissão de escrita
  • -x arquivo: existe e tem permissão de execução
  • -s arquivo: existe e tem tamanho maior que zero

Loops

Bash oferece várias estruturas de loop:

#!/bin/bash

# Loop for
echo "Loop for com lista:"
for nome in Ana João Pedro Maria; do
echo "Olá, $nome!"
done

# Loop for com sequência numérica
echo "Loop for numérico:"
for i in {1..5}; do
echo "Número $i"
done

# Loop while
echo "Loop while:"
contador=1
while [ $contador -le 5 ]; do
echo "Contador: $contador"
((contador++))
done

# Loop until (até que a condição seja verdadeira)
echo "Loop until:"
contador=5
until [ $contador -eq 0 ]; do
echo "Contagem regressiva: $contador"
((contador--))
done

# Loop for com comando find
echo "Arquivos no diretório atual:"
for arquivo in $(find . -type f -name "*.txt"); do
echo "Arquivo encontrado: $arquivo"
done

Funções

Funções permitem organizar e reutilizar código:

#!/bin/bash

# Definição de função
saudacao() {
echo "Olá, $1!"
echo "Bem-vindo ao script."
}

# Função com retorno
soma() {
local resultado=$(($1 + $2))
echo $resultado
return 0
}

# Chamada de função com argumento
saudacao "Usuário"

# Capturando retorno de função
resultado=$(soma 10 20)
echo "A soma é: $resultado"

# Função com variáveis locais
calcular_media() {
local a=$1
local b=$2
local media=$(( (a + b) / 2 ))
echo "A média de $a e $b é $media"
}

calcular_media 10 20
Boas Práticas
  • Use local para variáveis dentro de funções
  • Documente suas funções com comentários
  • Use valores de retorno para indicar sucesso (0) ou falha (não-zero)
  • Valide entradas para evitar comportamentos inesperados
  • Trate erros adequadamente

Exemplo de script completo com tratamento de erros:

#!/bin/bash

# Script para backup de arquivos
# Uso: ./backup.sh /caminho/origem /caminho/destino

# Função para exibir mensagem de erro e sair
erro() {
echo "ERRO: $1" >&2
exit 1
}

# Verificar número de argumentos
if [ $# -ne 2 ]; then
erro "Número incorreto de argumentos. Uso: ./backup.sh origem destino"
fi

origem="$1"
destino="$2"

# Verificar se origem existe
if [ ! -d "$origem" ]; then
erro "Diretório de origem não existe: $origem"
fi

# Verificar se destino existe, senão criar
if [ ! -d "$destino" ]; then
mkdir -p "$destino" || erro "Não foi possível criar o diretório de destino: $destino"
fi

# Nome do arquivo de backup
data=$(date +%Y%m%d_%H%M%S)
arquivo_backup="backup_$data.tar.gz"
caminho_completo="$destino/$arquivo_backup"

# Realizar backup
echo "Iniciando backup de $origem para $caminho_completo..."
tar -czf "$caminho_completo" -C "$(dirname "$origem")" "$(basename "$origem")" || erro "Falha ao criar arquivo tar"

# Verificar resultado
if [ -f "$caminho_completo" ]; then
tamanho=$(du -h "$caminho_completo" | cut -f1)
echo "Backup concluído com sucesso: $caminho_completo ($tamanho)"
exit 0
else
erro "Backup falhou: arquivo não foi criado"
fi

Ferramentas de Cybersegurança

Esta seção cobre ferramentas populares de cybersegurança, especialmente aquelas disponíveis no Kali Linux, uma distribuição focada em testes de penetração e auditoria de segurança.

Aviso de Uso Responsável

As ferramentas nesta seção são poderosas e devem ser usadas apenas em ambientes para os quais você tenha autorização explícita para testar. O uso destas ferramentas contra sistemas sem permissão pode violar leis locais e internacionais.

Ferramentas de Reconhecimento

O reconhecimento é a primeira fase de um teste de penetração, onde se coleta informações sobre o alvo.

Whois RECON

Consulta informações de registro de domínios.

whois example.com

Informações obtidas:

  • Proprietário do domínio
  • Servidores DNS
  • Datas de criação e expiração
  • Contatos administrativos e técnicos
Dmitry RECON

Ferramenta para coleta de informações diversas sobre hosts.

dmitry -winsepo resultado.txt example.com

Opções comuns:

  • -w: Whois
  • -i: Informações de netcraft
  • -n: Busca de subdomínios
  • -s: Scan de portas
  • -e: Busca de e-mails
  • -p: TCP port scan
  • -o: Salva output em arquivo
TheHarvester RECON

Coleta e-mails, subdomínios, hosts, nomes de funcionários, portas abertas e banners.

theHarvester -d example.com -l 500 -b all

Opções principais:

  • -d: Domínio para buscar
  • -l: Limite de resultados
  • -b: Fontes de busca (google, bing, linkedin, etc.)
  • -f: Salva resultados em arquivo HTML e XML
Recon-ng RECON

Framework completo para reconhecimento com módulos organizados.

recon-ng

Comandos básicos:

# Dentro do Recon-ng
marketplace search
marketplace install recon/domains-hosts/google_site_web
modules load recon/domains-hosts/google_site_web
options set SOURCE example.com
run

Scanners

Scanners são usados para identificar hosts, portas abertas, serviços e potenciais vulnerabilidades.

Nmap RECON

O "canivete suíço" de scanners de rede. Detecta hosts, portas, serviços, sistemas operacionais e mais.

nmap -sS -A -T4 192.168.1.0/24

Tipos comuns de scan:

  • -sS: SYN scan (rápido e discreto)
  • -sT: TCP connect scan (completo)
  • -sU: UDP scan
  • -sV: Detecção de versão de serviços
  • -O: Detecção de sistema operacional
  • -A: Scan agressivo (OS + versão + scripts + traceroute)
  • -p-: Todas as portas
  • -T4: Velocidade (0-5, 5 é mais rápido)
Nikto RECON

Scanner de vulnerabilidades para servidores web, identifica arquivos perigosos, softwares desatualizados e problemas comuns.

nikto -h http://example.com

Opções úteis:

  • -ssl: Usar SSL para conexão
  • -port 8080: Especificar porta
  • -output relatorio.html: Salvar resultados
  • -Tuning x: Realizar testes específicos
  • -evasion 1: Técnicas de evasão de detecção
OpenVAS RECON

Framework de scanners de vulnerabilidades com interface web.

gvm-start

Características:

  • Mais de 50.000 testes de vulnerabilidades
  • Escaneamento completo de redes e sistemas
  • Relatórios detalhados
  • Gerenciamento de vulnerabilidades
  • Atualizações regulares de feeds de segurança
WPScan RECON

Scanner de segurança específico para WordPress.

wpscan --url http://example.com --enumerate u

Opções de enumeração:

  • --enumerate u: Usuários
  • --enumerate p: Plugins
  • --enumerate t: Temas
  • --enumerate vp: Plugins vulneráveis
  • --api-token TOKEN: Usar API para mais dados de vulnerabilidades

Exploits e Frameworks

Ferramentas que auxiliam na exploração de vulnerabilidades descobertas durante os testes.

Metasploit Framework EXPLOIT

O mais completo framework para desenvolvimento e execução de exploits.

msfconsole

Estrutura:

  • Exploits: Código que tira proveito de vulnerabilidades
  • Payloads: Código a ser executado após exploração bem-sucedida
  • Auxiliares: Módulos para scanners, fuzzing, etc.
  • Encoders: Codificadores para evasão de detecção
  • Post: Módulos de pós-exploração

O Metasploit será discutido em detalhes em uma seção dedicada.

Searchsploit EXPLOIT

CLI para buscar no banco de dados do Exploit-DB.

searchsploit apache 2.4.7

Opções úteis:

  • -x ID: Examinar exploit específico
  • -m ID: Copiar exploit para diretório atual
  • -w: Mostrar URL do Exploit-DB
  • --exclude="tipo|plataforma": Excluir resultados indesejados
BeEF (Browser Exploitation Framework) EXPLOIT

Framework dedicado à exploração de navegadores web.

beef-xss

Funcionalidades:

  • Hooking de navegadores (infectar navegadores-alvo)
  • Interface de comandos e controle
  • Ampla variedade de módulos para interagir com navegadores infectados
  • Capacidade de pivoting para outros alvos na rede
  • Social engineering através do navegador

Nmap Avançado

O Nmap (Network Mapper) é uma ferramenta de código aberto para descoberta de rede e auditoria de segurança. Vamos explorar recursos avançados.

Tipos de Scan

SYN Scan (-sS)

O tipo de scan padrão, rápido e relativamente discreto.

nmap -sS 192.168.1.1

Como funciona:

  • Envia pacote SYN
  • Se receber SYN/ACK, porta está aberta
  • Se receber RST, porta está fechada
  • Não completa as conexões TCP
TCP Connect Scan (-sT)

Scan completo que estabelece uma conexão.

nmap -sT 192.168.1.1

Como funciona:

  • Completa o handshake TCP de 3 vias
  • Mais detectável em logs
  • Mais confiável em alguns casos
  • Usado por default quando não se tem privilégios
UDP Scan (-sU)

Verifica portas UDP abertas, que são frequentemente ignoradas.

nmap -sU 192.168.1.1

Como funciona:

  • Envia pacotes UDP para as portas alvo
  • "ICMP port unreachable" significa porta fechada
  • Resposta ou ausência sugere porta aberta/filtrada
  • Geralmente lento, combine com --top-ports
FIN, NULL, XMAS Scans (-sF, -sN, -sX)

Scans furtivos para contornar firewalls e detecção.

nmap -sF 192.168.1.1

Como funcionam:

  • FIN: Envia pacote com flag FIN
  • NULL: Envia pacote sem flags
  • XMAS: Envia pacote com flags FIN, PSH e URG
  • RST para portas fechadas, sem resposta para abertas/filtradas
  • Não funciona bem contra sistemas Windows
Detecção de Versão (-sV)

Determina serviços e versões em portas detectadas.

nmap -sV --version-intensity 9 192.168.1.1

Opções adicionais:

  • --version-intensity: Define agressividade (0-9)
  • --version-light: Rápido (intensidade 2)
  • --version-all: Testes completos (intensidade 9)
  • --version-trace: Mostra testes sendo realizados
OS Detection (-O)

Tenta identificar o sistema operacional do alvo.

nmap -O --osscan-guess 192.168.1.1

Opções relacionadas:

  • --osscan-limit: Só tenta identificar alvos promissores
  • --osscan-guess: Mostrar palpites agressivos
  • --max-os-tries: Limita número de tentativas

Scripts NSE (Nmap Scripting Engine)

NSE estende as funcionalidades do Nmap com scripts Lua para diversas tarefas.

nmap --script=vuln 192.168.1.1

Categorias principais de scripts:

Categoria Descrição Exemplo
default Scripts padrão executados com -sC nmap -sC 192.168.1.1
safe Scripts não intrusivos --script=safe
vuln Detecta vulnerabilidades --script=vuln
exploit Tenta explorar vulnerabilidades --script=exploit
auth Tenta burlar autenticação --script=auth
brute Força bruta em credenciais --script=brute
discovery Coleta informações adicionais --script=discovery
dos Scripts que podem causar DoS --script=dos
intrusive Scripts que podem impactar o alvo --script=intrusive
malware Detecta backdoors e malware --script=malware

Exemplos de scripts NSE úteis:

nmap --script=http-title 192.168.1.1
nmap --script=ssl-enum-ciphers --script-args=vulns.showall 192.168.1.1 -p 443
nmap --script=smb-os-discovery 192.168.1.1
nmap --script=dns-zone-transfer --script-args=dns-zone-transfer.domain=example.com -p 53 ns.example.com

Técnicas de Evasão

Opções para tornar os scans menos detectáveis por sistemas de segurança.

Fragmentação (-f)

Divide pacotes TCP em fragmentos menores.

nmap -f 192.168.1.1

Use -ff para fragmentos ainda menores.

Decoys (-D)

Esconde origem real entre vários IPs de origem falsos.

nmap -D 10.0.0.1,10.0.0.2,ME 192.168.1.1

"ME" representa seu próprio IP na lista de decoys.

Spoofing (-S)

Falsifica o endereço IP de origem (requer acesso privilegiado).

nmap -S 10.0.0.1 192.168.1.1

Útil principalmente para determinar regras de firewall.

Timing (-T)

Controla a velocidade do scan para ser menos detectável.

nmap -T0 192.168.1.1

Templates de timing:

  • -T0 (Paranóico): Muito lento, um porto por vez
  • -T1 (Furtivo): Lento, detecta menos IDS
  • -T2 (Educado): Lento, usa menos recursos
  • -T3 (Normal): Comportamento padrão
  • -T4 (Agressivo): Rápido em redes confiáveis
  • -T5 (Insano): Muito rápido, pode perder informações

Scan completo para auditoria:

nmap -sS -sU -T4 -A -v -PE -PP -PS80,443 -PA3389 -PU40125 -PY --script "default or (discovery and safe)" 192.168.1.0/24

Wireshark

O Wireshark é um analisador de protocolo de rede usado para captura e inspeção interativa do tráfego em uma rede.

Capturas

Para iniciar o Wireshark com interface gráfica:

wireshark

Para capturas via linha de comando:

tshark -i eth0 -w captura.pcap
Dicas para Captura
  • Use o modo promíscuo para capturar todo o tráfego na rede
  • Utilize filtros de captura para reduzir o volume de dados
  • Para redes com switch, considere técnicas como port mirroring ou uso de hub
  • Em VLANs, configure a interface para capturar a VLAN correta
  • Em redes sem fio, use o modo monitor para capturar todo o tráfego

Filtros

Wireshark utiliza dois tipos de filtros:

Filtros de Captura

Aplicados durante a captura para reduzir a quantidade de pacotes.

Utilizados no campo "Capture Filter" ou com opção -f no tshark.

Exemplos:

  • host 192.168.1.1: Tráfego de/para um host
  • net 192.168.0.0/24: Tráfego de/para uma rede
  • port 80: Tráfego para/de porta 80
  • tcp port 80 or tcp port 443: HTTP ou HTTPS
  • not arp: Exclui tráfego ARP
Filtros de Exibição

Aplicados após a captura para filtrar os pacotes mostrados.

Utilizados no campo "Display Filter" na interface.

Exemplos:

  • ip.addr == 192.168.1.1: Pacotes de/para um IP
  • http: Tráfego HTTP
  • tcp.port == 80 or tcp.port == 443: HTTP ou HTTPS
  • http.request.method == "POST": Requisições POST
  • tcp.flags.syn == 1: Pacotes SYN
  • dns.qry.name contains "google": Consultas DNS com "google"
  • !(arp or dns or icmp): Exclui ARP, DNS e ICMP
tshark -i eth0 -f "port 80" -Y "http.request.method == GET"

Análise de Pacotes

Tipo de Análise Filtros/Ferramentas
Análise de fluxo TCP Statistics > Flow Graph, ou
tcp.stream eq 0 para ver um fluxo específico
Extrair arquivos HTTP File > Export Objects > HTTP
Análise de credenciais http.request.method == "POST" ou
ftp.request.command == "PASS"
Problemas de rede tcp.analysis.flags para ver retransmissões,
duplicados e outros problemas
Análise de latência Statistics > TCP Stream Graph > Round Trip Time
Análise de SSL/TLS ssl ou tls, Configure decriptação
com Edit > Preferences > Protocols > TLS

Comandos tshark para análise:

tshark -r captura.pcap -Y "http" -T fields -e http.host -e http.request.uri | sort | uniq
tshark -r captura.pcap -Y "dns" -T fields -e dns.qry.name | sort | uniq -c | sort -nr
tshark -r captura.pcap -Y "http.request.method == POST" -T fields -e frame.time -e ip.src -e http.host -e http.request.uri

Hydra

O Hydra é uma ferramenta para testes de força bruta em credenciais, suportando diversos protocolos como HTTP, FTP, SSH, entre outros.

Uso Ético

O Hydra é uma ferramenta poderosa que deve ser usada apenas em sistemas onde você tem permissão explícita para testar. Uso não autorizado pode ser ilegal e antiético.

Ataques de Força Bruta

Exemplos de uso em diferentes protocolos:

hydra -l admin -P /usr/share/wordlists/rockyou.txt ssh://192.168.1.1
hydra -l admin -P /usr/share/wordlists/rockyou.txt 192.168.1.1 http-post-form "/login.php:username=^USER^&password=^PASS^:F=Login failed"

Opções comuns:

Opção Descrição

-L arquivo Arquivo com lista de nomes de usuário
-p senha Especifica uma única senha
-P arquivo Arquivo com lista de senhas
-C arquivo Arquivo com pares login:senha

Wordlists de qualidade são essenciais para testes de força bruta eficazes.

Geração de Wordlists

Ferramentas para gerar wordlists customizadas:

  • Crunch: Gera listas baseadas em padrões
  • CeWL: Extrai palavras de sites para wordlists
  • John the Ripper: Modo de regras para expansão de wordlists
  • Cupp: Cria wordlists baseadas em perfis

Exemplo de ataque completo com Hydra:

# 1. Coletar informações sobre formulário de login

# 2. Testar manualmente para verificar mensagens de erro

# 3. Gerar wordlist baseada no site
cewl -d 2 -m 6 -w wordlist.txt http://example.com
cat /usr/share/wordlists/rockyou.txt >> wordlist.txt

# 4. Executar Hydra com limite de threads e saída detalhada
